The pipeline

From camera to sorted bin

Four stages, running continuously. The arm returns to its home position between each pick so the camera always has a clear view for the next one.

01
👁️
Detect
YOLO OBB identifies and classifies each piece of plastic. Only objects seen in 5 consecutive frames make it through.
02
📍
Localise
Depth data and a calibrated homography matrix convert pixel positions into millimetre-accurate robot-frame coordinates.
03
🧭
Plan
MoveIt2 plans a collision-free path. It tries eight variations and picks whichever one moves the joints the least.
04
📦
Sort
The gripper descends, closes on the object, and lifts. Stall detection confirms a real grasp. The arm drops the piece in the right bin.
Machine Learning

Trained to see rotated objects

Standard object detectors give you a rectangle. That's not enough when you need to know which way to rotate the gripper. We use an Oriented Bounding Box model that returns the object's angle alongside its class — so the arm always approaches aligned with the long axis.

The model was trained on three classes found in typical household recycling: HDPE bottles, PET bottles, and PP containers. Before any detection reaches the motion planner it has to appear stably in five frames in a row — cutting out flickers and reflections.

hdpe_bottle pet_bottle pp_container conf ≥ 0.70 5-frame stability
Perception

Pixels to millimetres

A RealSense D435 gives us aligned colour and depth at 30 fps. The depth alone isn't enough — camera-to-robot calibration is what makes the numbers trustworthy.

We place ArUco markers at known positions on the table and solve for a homography matrix that maps any pixel directly to a robot XY coordinate. Depth adds the Z. A fitting-plane equation — calibrated over the whole work surface — corrects for any tilt so every pick lands where expected.

11×11 median kernel
Depth is sampled from 121 pixels around the centroid and median-filtered to reject noise and specular spikes.
5-frame smoother
XY position is averaged across the last five readings so the commanded pick location doesn't jitter.
Homography fallback
If homography fails, a 3D ray is cast through the pixel and intersected with the table plane.
ROS bridge
Annotated frames are published on /camera/annotated so the GUI shows a live feed.
Motion Planning & Control

Getting there without hitting anything

MoveIt2 handles the hard part: finding a path from the current joint configuration to the grasp pose that doesn't clip the table, the camera tripod, or the object being picked. The planner tries eight different solutions and runs with whichever takes the smallest total joint movement — avoiding the annoying 270° shoulder swings that technically reach the same point.

The final descent and raise are Cartesian — the gripper moves in a straight vertical line so it doesn't knock adjacent objects. A background thread watches the gripper during transport and flags a drop if the fingers close all the way unexpectedly.

RRTConnect Cartesian descent DropMonitor 500 Hz RTDE 3× retry
Interaction & Execution

One screen, full control

The GUI gives the operator everything in one place: a live annotated camera feed, what objects are currently detected and where, the robot's current state, and the controls to start, stop, or intervene at any moment.

Every button publishes a plain string to a ROS topic — keeping the interface completely decoupled from the motion logic. ESTOP halts the arm mid-trajectory. Bin poses are taught by jogging the arm and pressing Save rather than typing coordinates.

State machine
IDLE → WAITING → PREGRASP → GRASPING → TRANSPORTING → DONE, shown live in the GUI.
Bin teaching
Jog the arm to any bin position, press Save — coordinates written to bin_poses.json immediately.
Class filter
Pick a plastic type in the GUI and the robot clears all of that class before moving to others.
Sim mode
Launch with sim:=true to run the full sequence without a physical object — gripper skips stall detection.
The workspace

A purpose-built sorting surface

Everything Sort-O sees and touches is contained within a 500 × 320 mm black board mounted on the trolley. The dark surface gives the camera clean contrast against plastic objects and keeps them from sliding around during picks.

Four ArUco markers — one at each corner, numbered 1 to 4 — act as the calibration reference. When the camera-to-robot homography is computed, these markers define exactly where the robot coordinate frame maps onto the image plane. A custom plywood bin assembly clips onto the trolley handle with three square openings, one per plastic type.

Board
500 × 320 mm black surface on the trolley. Constrains the pick zone and gives depth camera contrast.
ArUco markers
4 markers at board corners (IDs 1–4). Used once to calibrate the camera-to-robot homography matrix.
Camera mount
Tripod positioned to look straight down at the board. Fixed height — moving it requires recalibration.
Bin assembly
3-slot plywood frame clipped to the trolley handle. Each slot is a named drop-off point for one plastic class.

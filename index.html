Overview

How Sort-O Works

Four tightly-coupled subsystems work together to turn a pile of mixed plastic into sorted bins — fully autonomously.

STEP 01
🎯
Detect
YOLO OBB model identifies and classifies plastic objects from the RealSense camera feed at 10 Hz with a stability filter.
STEP 02
📍
Localise
Depth data + homography calibration maps each detection from pixel coordinates to a precise 3D pose in the robot's world frame.
STEP 03
🦾
Plan
MoveIt2 plans a collision-free trajectory using RRTConnect, selecting the lowest-cost path from multiple attempts.
STEP 04
✅
Sort
The RG2 gripper grasps the object using stall-detection feedback, lifts it to safe height, and places it in the correct bin.
Machine Learning

YOLO OBB Object Detection

A custom-trained YOLOv8 Oriented Bounding Box model identifies plastic objects with rotation awareness — critical for gripper alignment.

  • Oriented bounding boxes capture object yaw angle, enabling precise top-down grasp orientation
  • Three trained classes: hdpe_bottle, pet_bottle, pp_container
  • Confidence threshold of 0.70 filters low-quality detections before depth processing
  • Stability filter requires 5 consecutive detections before a pick is triggered
  • Per-instance tracking with persistent IDs across frames (e.g. pet_bottle_0)
🧠
OBB Model
YOLOv8 OBB architecture. Outputs class, confidence, centre pixel, and rotation angle for each detection.
🎲
Stability Filter
Only publishes objects seen in ≥ 5 consecutive frames. Eliminates false triggers from single-frame glitches.
🔄
Track Matching
Each detection is matched to the nearest existing track within 60 px so IDs stay consistent as the arm moves.
📐
Angle Extraction
OBB rotation angle is converted to a gripper yaw command, aligning fingers with the object's long axis.
📷
RealSense D435
640 × 480 aligned colour + depth at 30 fps. Depth aligned to colour so each pixel has a direct depth reading.
🗺️
Homography Calibration
ArUco markers map pixel XY → robot XY using a 3×3 homography matrix. Eliminates perspective distortion.
📏
Depth Smoothing
11 × 11 px median kernel + 50 mm spike filter + 5-frame rolling average suppress depth noise.
📡
ROS Image Bridge
Annotated frames published to /camera/annotated so the GUI can display a live detection feed.
Perception

3D Object Localisation

Raw pixels become millimetre-accurate 3D poses in the robot's coordinate frame through a calibrated depth-to-world pipeline.

  • Primary: homography matrix maps object centroid pixel to robot XY (mm)
  • Fallback: 3D ray–table plane intersection using the 4×4 transform matrix
  • Z computed from fitted plane equation calibrated over the work surface
  • Class-specific grip offsets account for object height (HDPE vs PET vs PP)
  • Outputs published to perception/objects as object_msgs/ObjectArray

Motion Planning & Control

MoveIt2 Trajectory Planning

MoveIt2 turns 3D object poses into safe, collision-free arm trajectories — replanning reactively after each pick to maintain a clear camera view.

  • RRTConnect planner with 8 attempts per goal; executes lowest-cost trajectory to avoid unnecessary joint rotation
  • Collision objects (table, camera tripod, grasped object) kept live in the MoveIt planning scene
  • Cartesian path for precise vertical descend and raise during grasp
  • IterativeParabolicTimeParameterization ensures smooth velocity profiles within hardware limits
  • DropMonitor background thread detects object drops during transport via gripper re-poll
  • Repeat-fail guard skips a location after 3 consecutive grasp failures
🤖
UR3e Arm
6-DOF arm with 500 mm reach. Controlled at 500 Hz via RTDE over Ethernet through ur_ros2_driver.
✊
OnRobot RG2
Parallel gripper with 110 mm span. Stall-detection feedback confirms object contact — no force sensor needed.
🛡️
Collision Scene
Ground plane, camera tripod, and each picked object are added as MoveIt collision objects before planning.
🔁
Reactive Loop
Returns home after each pick to restore the camera view, then waits for the next fresh perception message.
🖥️
Status GUI
Live tkinter dashboard showing robot state, detected objects, camera feed, and bin queue.
⏹️
ESTOP
Hardware-safe emergency stop published to /motion_system/command — halts arm mid-trajectory immediately.
📦
Bin Teaching
Manually jog the arm to a bin position, press Save Bin in the GUI — pose is persisted to bin_poses.json.
🎯
Class Priority
GUI lets you target a specific plastic type so Sort-O picks all of that class before moving to others.
Interaction & Execution

GUI & Command Interface

An operator GUI connects the human to every part of the system — start/stop, live camera feed, bin management, and real-time status.

  • All commands sent via ROS topic /motion_system/command — decoupled from the motion controller
  • Live annotated camera feed from /camera/annotated displayed in the GUI sidebar
  • Robot state broadcast on /robot_state: IDLE → WAITING → PREGRASP → GRASPING → TRANSPORTING → DONE
  • Simulation mode (sim:=true) disables stall detection for testing without a physical object
  • Single bringup command launches all four subsystems with staggered delays

Setup & Launch Guide

Everything you need to get Sort-O running — from plugging in the camera to pressing Start in the GUI.

📷 Camera — Intel RealSense D435

📷
Intel RealSense D435

Depth + RGB camera. Depth is aligned to the colour frame so every pixel has a matching depth reading. Must be connected via USB 3 — USB 2 will work but depth quality and frame rate drop significantly.

PropertyValue
Colour resolution640 × 480 px
Depth resolution640 × 480 px (aligned to colour)
Frame rate30 fps
Colour formatBGR8
Depth formatZ16 (millimetres)
ConnectionUSB 3.x (blue port)
Mount positionTripod, looking straight down at workspace board
ROS driverpyrealsense2 (Python, direct SDK)
Moving the camera after calibration means the homography matrix is wrong. Recalibration is required — run obb_detector_node, place all 4 ArUco markers, and save a new camera_to_robot_calibration.json.

1 Prerequisites

Before running anything, make sure these are installed:

Ubuntu 22.04 — install once
# ROS 2 Humble
sudo apt install ros-humble-desktop ros-humble-ur

# MoveIt 2
sudo apt install ros-humble-moveit

# Python dependencies
pip install ultralytics pyrealsense2 opencv-python scipy numpy scikit-learn pillow

# Gripper serial comms
sudo apt install libnet1-dev
Clone & build (once)
git clone https://github.com/Jun-Je0ng/Robotics-Studio-2.git
cd Robotics-Studio-2

# Install ROS dependencies
vcs import src --input src/ur_onrobot/required.repos --recursive
rosdep install -y --from-paths src --ignore-src

# Build
colcon build --symlink-install
source install/setup.bash

2 Teach pendant — External Control setup

This is a one-time setup. The UR robot needs to know your laptop's IP so it can connect back to ROS after you press Play.

A — Configure Tool I/O for the gripper

1️⃣ On the teach pendant go to Installation → General → Tool I/O
⚙️ Set Controlled by: User  ·  Tool Output Voltage: 24V
📡 Communication Interface: Baud Rate 1M · Parity Even · Stop Bits One · RX Idle Chars 1.5 · TX Idle Chars 3.5
🔌 Standard Output: Digital Output 0 & 1 → Sinking (NPN)
🔗 Still in Installation, scroll to URCaps → OnRobot → set connection to "No Connection"

B — Set up External Control

🌐 Check your laptop's IP: run hostname -I in a terminal — note the 192.168.x.x address
📋 On pendant: Installation → URCaps → External Control → set Host IP to your laptop's IP
📝 Go to Program tab → URCaps → add External Control to the program list
▶️ Press Play — the pendant will wait for ROS to connect. You'll see "Robot connected to reverse interface" in the driver terminal.
Start the ROS robot driver before pressing Play. The driver opens a port and waits; the pendant connects to it when Play is pressed.

URSim (simulator, no physical robot)

If testing without the real robot, use URSim via Docker. The process is the same but the robot IP is 192.168.56.101 and the Host IP in External Control should be 192.168.56.1 (your VirtualBox adapter).

Start URSim
ros2 run ur_client_library start_ursim.sh -m ur3e
# Then open http://192.168.56.101:6080/vnc.html in a browser

3 Build the workspace

Run this once after cloning, and again whenever you change a C++ file. Python nodes don't need a rebuild.

Terminal
cd ~/git/Robotics-Studio-2
colcon build --symlink-install
source install/setup.bash
--symlink-install means Python files (GUI, perception, translator) are linked rather than copied — edits take effect immediately on next run without rebuilding.

4 Run the system

Each step runs in its own terminal. Source the workspace in each one first.

One-command shortcut: ros2 launch ur_gripper_demo bringup.launch.py launches all four steps automatically with staggered delays. Use the separate terminals below when you need to see individual logs or debug a specific node.
1
Robot driver
Connects to the UR3e over Ethernet and starts the 500 Hz RTDE control loop.
Terminal 1
source install/setup.bash
ros2 launch ur_onrobot_control start_robot.launch.py \
    ur_type:=ur3e \
    onrobot_type:=rg2 \
    robot_ip:=192.168.0.197 \
    launch_rviz:=false

# For URSim: robot_ip:=192.168.56.101

Wait for: Robot connected to reverse interface. Ready to receive control commands.

Then press ▶ Play on the teach pendant.

2
MoveIt + motion controller
Starts MoveIt2, loads the URDF, and launches the reactive pick-place node.
Terminal 2
source install/setup.bash
ros2 launch ur_gripper_demo ur_moveit.launch.py \
    ur_type:=ur3e \
    demo_type:=reactive \
    sim:=false

# Add sim:=true to skip gripper stall detection (testing without objects)
3
Camera + OBB detector
Opens the RealSense, runs YOLO inference, and publishes stable detections. An OpenCV window shows the annotated live feed.
Terminal 3
source install/setup.bash
ros2 run plastic_detection obb_detector_node

Status shows Stabilising... (orange) until 5 consecutive frames, then PUBLISHING (green).

4
GUI
Opens the operator control panel with live camera feed, detection list, and Start/ESTOP controls.
Terminal 4
source install/setup.bash
ros2 run ur_gripper_demo status_gui

Verify each node is publishing

Check topics (any terminal)
# Raw detections from the camera node
ros2 topic echo /plastic_detections

# Structured messages going to the motion controller
ros2 topic echo /perception/objects

# Commands sent by the GUI
ros2 topic echo /motion_system/command

# Robot state broadcast
ros2 topic echo /robot_state

5 Teaching bin poses

Before pressing Start, the robot needs to know where each bin is. Do this once per session (or when bins move).

🕹️ On the teach pendant, enable Freedrive and move the arm to the drop position above a bin
💾 In the GUI, type the bin name in the Save Bin field and press Save Bin — e.g. hdpe_bottle, pet_bottle, pp_container
✅ Repeat for each bin. Poses are written to config/bin_poses.json immediately.
You can also save from a terminal: ros2 topic pub --once /motion_system/command std_msgs/msg/String "data: 'SAVE_BIN:pet_bottle'"

6 Simulation mode

Pass sim:=true to the MoveIt launch to disable gripper stall detection. The robot moves through the full pick-place sequence and assumes every grasp succeeds — useful for testing arm motion and GUI controls without needing a physical object under the camera.

Terminal 2 — sim mode
ros2 launch ur_gripper_demo ur_moveit.launch.py \
    ur_type:=ur3e \
    demo_type:=reactive \
    sim:=true
With sim:=true the gripper will attempt to close on nothing and move to the bin regardless. Don't use this on the real robot with delicate objects nearby.

Test GUI buttons without the robot

Open the GUI on its own and inject fake status messages from a second terminal to verify the display updates correctly:

Inject fake status
# Fake robot state
ros2 topic pub --once /robot_state std_msgs/msg/String "{data: 'GRASPING'}"

# Watch what commands the GUI sends when you click buttons
ros2 topic echo /motion_system/command

Setup & Launch Guide

Everything you need to get Sort-O running — from plugging in the camera to pressing Start in the GUI.

📷
Camera — Intel RealSense D435

The RealSense D435 provides aligned colour and depth frames. Every pixel in the colour image has a matching depth value, which is what lets the detector compute 3D object positions. Connect it to a USB 3 (blue) port — USB 2 drops frame rate and depth quality noticeably.

PropertyValue
Colour resolution640 × 480 px
Depth resolution640 × 480 px — aligned to colour
Frame rate30 fps
Colour formatBGR8
Depth formatZ16 (millimetres)
ConnectionUSB 3.x (blue port)
MountTripod — looking straight down at workspace board
ROS driverpyrealsense2 Python SDK — no ROS camera node needed
⚠️ Moving the camera after calibration invalidates the homography matrix. If the camera shifts, run calibration again before starting the system.
1
Prerequisites

Run these once on a fresh Ubuntu 22.04 install.

System dependencies
# ROS 2 Humble + UR driver
sudo apt install ros-humble-desktop ros-humble-ur

# MoveIt 2
sudo apt install ros-humble-moveit

# Gripper serial comms
sudo apt install libnet1-dev

# Python packages
pip install ultralytics pyrealsense2 opencv-python scipy numpy scikit-learn pillow
Clone & build
git clone https://github.com/Jun-Je0ng/Robotics-Studio-2.git
cd Robotics-Studio-2

# Pull driver dependencies
vcs import src --input src/ur_onrobot/required.repos --recursive
rosdep install -y --from-paths src --ignore-src

# Build
colcon build --symlink-install
source install/setup.bash
2
Camera calibration

Calibration maps the camera's pixel coordinates to the robot's world frame. It must be done once after first setup, and again any time the camera or workspace board is moved.

Place all four ArUco markers at the corners of the workspace board (numbered 1–4), make sure the camera can see all of them, then run:

Navigate to the calibration script
cd ~/git/Robotics-Studio-2/src/plastic_detection/plastic_detection/
python3 camera_calibration.py

The script detects the ArUco markers, computes the homography matrix, and writes the result to camera_to_robot_calibration.json. You should see all 4 markers detected — if any are missing, reposition them and rerun.

💡 Keep the workspace lighting consistent between calibration and normal operation. Shadows over the ArUco markers can cause detection to fail.
3
Teach pendant — robot setup

Two things to configure on the pendant. Both are one-time setups that persist between sessions.

Part A — Tool I/O (for the RG2 gripper)
1
Navigate to Installation → General → Tool I/O
2
Set Controlled by to User and Tool Output Voltage to 24V
3
Under Communication Interface, configure:
Baud Rate: 1M  ·  Parity: Even  ·  Stop Bits: One  ·  RX Idle Chars: 1.5  ·  TX Idle Chars: 3.5
4
Under Standard Output, set both Digital Output 0 and Digital Output 1 to Sinking (NPN)
5
Still in Installation, scroll to URCaps → OnRobot and set the connection to No Connection
Part B — External Control (connects robot to ROS)
ℹ️ Find your laptop's IP first: run hostname -I in a terminal and note the 192.168.x.x address.
1
Go to Installation → URCaps → External Control
Set Host IP to your laptop's IP address
2
Go to the Program tab → open URCaps → add External Control to the program tree
3
Start the ROS robot driver (Terminal 1 below), then press ▶ Play on the pendant
You'll see Robot connected to reverse interface appear in the driver terminal once connected.
URSim — testing without the physical robot

Start the simulator, then follow Part B above using 192.168.56.101 as the robot IP and 192.168.56.1 as the Host IP.

Start URSim
ros2 run ur_client_library start_ursim.sh -m ur3e
# Open the Polyscope interface at: http://192.168.56.101:6080/vnc.html

Build the workspace

Run once after cloning, and again whenever a C++ file changes. Python nodes don't need a rebuild thanks to --symlink-install.

Terminal
cd ~/git/Robotics-Studio-2
colcon build --symlink-install
source install/setup.bash

Run the system

Open four terminals. Source the workspace at the top of each one.

⚡ One-command shortcut: ros2 launch ur_gripper_demo bringup.launch.py launches all four nodes automatically with staggered delays. Use separate terminals when you want individual logs.
1
Robot driver
Connects to the UR3e and starts the 500 Hz RTDE control loop
Terminal 1
source install/setup.bash
ros2 launch ur_onrobot_control start_robot.launch.py \
    ur_type:=ur3e \
    onrobot_type:=rg2 \
    robot_ip:=192.168.0.197 \
    launch_rviz:=false

# URSim: use robot_ip:=192.168.56.101 instead
Wait for: Robot connected to reverse interface. Ready to receive control commands.
Then press ▶ Play on the teach pendant.
2
MoveIt + motion controller
Starts MoveIt2 and the reactive pick-place node
Terminal 2
source install/setup.bash
ros2 launch ur_gripper_demo ur_moveit.launch.py \
    ur_type:=ur3e \
    demo_type:=reactive \
    sim:=false

# Use sim:=true to skip stall detection when testing without objects
3
Camera + OBB detector
Opens the RealSense and runs YOLO inference. An OpenCV window shows the live annotated feed.
Terminal 3
source install/setup.bash
ros2 run plastic_detection obb_detector_node
Status shows Stabilising... until 5 consecutive frames, then turns PUBLISHING when detections are stable.
4
GUI
Opens the operator panel — live camera feed, detection list, Start / ESTOP controls
Terminal 4
source install/setup.bash
ros2 run ur_gripper_demo status_gui
6
Teaching bin poses

Before pressing Start, the robot needs to know where each bin is. Do this once (poses are saved to disk and persist between sessions).

🕹️
Enable Freedrive on the teach pendant and manually move the arm to the drop position above a bin
💾
In the GUI, enter the bin name and press Save Bin. Use the class names exactly: hdpe_bottle, pet_bottle, or pp_container
🔁
Repeat for each bin. Poses are written to config/bin_poses.json immediately and loaded automatically on next launch
💡 You can also save from a terminal: ros2 topic pub --once /motion_system/command std_msgs/msg/String "data: 'SAVE_BIN:pet_bottle'"
7
Simulation mode

Pass sim:=true to the MoveIt launch to disable gripper stall detection. The robot runs the full pick-place sequence and assumes every grasp succeeds — useful for testing arm motion, GUI controls, and bin poses without needing a physical object under the camera.

Terminal 2 — sim mode
ros2 launch ur_gripper_demo ur_moveit.launch.py \
    ur_type:=ur3e \
    demo_type:=reactive \
    sim:=true
⚠️ With sim:=true the arm will move to the bin even with nothing in the gripper. Keep the workspace clear of obstacles when testing.

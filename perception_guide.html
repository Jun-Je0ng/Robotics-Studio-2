1See It — Camera Setup & Alignment

The RealSense is two cameras in one body — an RGB sensor for colour and a separate depth sensor for distance. Because they sit a few centimetres apart, they see slightly different angles. Without alignment, the depth pixel at (300, 200) would belong to a different real-world point than the colour pixel at (300, 200). rs.align warps the depth image to sit exactly on top of the colour image so every pixel matches.

2Recognise It — YOLO OBB Detection

A standard bounding box is always axis-aligned — useless for knowing which way a bottle is pointing. Oriented Bounding Box (OBB) draws a rotated rectangle that fits tightly at any angle and returns that angle. The model was trained on images of your specific plastic types. Anything below 70% confidence is dropped before any further processing.

3Locate It — Depth, Deprojection & Robot Transform

This is the core of perception — turning a 2D pixel into a 3D coordinate the robot arm can move to. Three sub-steps: read the depth at the detection centre (using a median to reject noise), convert pixel + depth into a 3D camera-frame point, then multiply by a pre-calibrated 4×4 matrix to land it in the robot's coordinate system. The matrix was computed by placing 4 ArUco markers at known robot positions and solving for the transform.

# inside get_median_depth (lines 67-72):

valid  = region[region > 0]            # throw away zero/invalid pixels
return float(np.median(valid)) / 1000.0   # median of good pixels, mm → metres

# line 274 — apply calibration matrix: camera frame → robot base_link frame

4Trust It — Smoothing & Stability Gating

Even with good depth, the 3D position still jitters a few millimetres frame-to-frame — just sensor noise. Temporal smoothing fixes this by averaging the last 5 readings. Stability gating solves a different problem: a shadow, reflection, or passing hand can fool YOLO for one frame. You wait for 5 consecutive detections before telling the robot anything. One missed frame resets the counter to zero.

5Describe It — Dimensions, Height & Orientation

Position alone is not enough — the robot needs to know how big the object is (to set gripper width), how tall it is (to know how deep to reach), and which way it is facing (to align the gripper fingers along the long axis). Height is measured by comparing the depth of the table surface just outside the bounding box against the depth of the object top — the difference is the physical height. Orientation comes from the OBB angle, but it must be negated because the image Y-axis points downward while the robot Y-axis points forward — they are mirrored.

The Whole Story — One Paragraph The RealSense camera gives you two images every frame — colour and depth — which are physically offset from each other and must be aligned before anything else. Once aligned, the colour image goes into a YOLO OBB model which draws a tight rotated box around each plastic object and tells you its class, confidence, pixel position, and angle. The depth value at that pixel is read using a median kernel to reject noise, then filtered again to remove sudden spikes between frames. That depth plus the pixel position is deprojected into a 3D point in the camera's coordinate space, then multiplied by a pre-calibrated 4×4 matrix to land it in the robot's coordinate space. The position is smoothed over 5 frames to kill jitter, and the detection must appear in 5 consecutive frames before anything is published — so reflections and false positives never reach the robot. On top of position, the system also computes the object's real-world length and width by scaling pixels using the focal length, measures its height from the depth difference between the table and the object top, and converts the rotation angle into a quaternion — negated to fix the Y-axis mirror between camera and robot. The result is a complete 3D description — position, orientation, and dimensions — published over ROS for the robot to act on.
